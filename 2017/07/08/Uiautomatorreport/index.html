Uiautomator 2.0 测试报告如何生成？

发表于 2017-07-08 | 分类于 自动化
你学会了uiautomator后，终于可以和小B一样，在相同的时间内轻松的完成领导安排的任务了。但是，小B却受到了老板的夸赞，这是为什么呢？因为小B不仅写完了自动化，还生成了一份漂亮的自动化测试报告用邮件发送给了老板，让老板看到了他的劳动成果～～～你啊，没事，别灰心，我现在就来教你～～

写在前面的话

我们知道，在执行自动化测试用例的时候，控制台会输出各种信息，有错误的时候也会在控制台报错。这样的信息无论是对你还是对团队而言，都不是有效又直观的。我们需要的是一份测试报告，这份测试报告不仅要内容详细，格式美观，还要让除你之外的其他人看到能一眼就能明白你此次测试的一个结果。所以有一份漂亮的测试报告很重要！
有了这样的想法后，我开始调研好用的日志框架。
市面上最受欢迎的Java日志组件可能要属log4j了，log4j灵活丰富的配置可以帮助开发人员全方位地掌控日志信息。然而对于软件自动化测试来说，无论是log4j，还是Junit 、testNG等测试框架下自动生成的日志和报告，对结果统计和数据展现的表现力就没那么优秀了，但是log4j它详尽的log信息又是我们需要的，鉴于此我在这里将log4j和拥有图表统计的extentReports日志框架结合在一起，从而满足对于不同的需求，选择不同的日志框架。
打算将它们结合起来打造出一份美丽的测试报告，但是最终以失败告终，因为extentReports日志框架不支持android项目，没有办法，只有想别的方法。
确实，自己之前对Uiautomator2.0也并不是很了解，翻看资料后，才知道其实U2自带了测试报告，并且颜值并不差，废话不多说，下面来见识一下。

生成测试报告

上一篇文章中，我们提到执行测试用例，只需要右键然后runXXX就好了，但是这样是没有办法生成测试报告的。
要想生成测试报告，我们需要在命令行运行./gradlew cC，这样你的报告会生成在项目名\app\build\reports\androidTests\connected下面，另外，每次执行的时候会覆盖之前的报告，如果不想报告覆盖，你可以在gradle里加入
android {
compileSdkVersion 25
buildToolsVersion “25.0.3”
testOptions {
reportDir=”$project.buildDir”+releaseTime()
}
// 获取当前系统时间
def releaseTime() {
return new Date().format(“yyyy-MM-dd-HH-mm”, TimeZone.getTimeZone(“UTC”))
}
这样的话！！！报告就生成了！！！就是这么简单！！！效果如下图所示：
